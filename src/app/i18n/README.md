# i18n в приложении

## Статичные строки (интерфейс)

Все подписи кнопок, заголовки, плейсхолдеры и сообщения берутся из `resources.ts` и переводятся через `useTranslation()` / `t('key')`. Добавляйте новые ключи в `resources.ru`, `resources.en`, `resources.kg`.

## Контент из БД (названия книг, авторы)

**Текущее поведение:** названия книг, имена авторов и другой контент приходят с бэкенда в одном языке (как хранится в БЗ). Через `t()` их не прогоняем — i18n рассчитан на ключи из кода, а не на произвольный текст из API.

**Как можно переводить контент из БЗ:**

1. **Локализованные поля в API** (предпочтительно)  
   Бэкенд отдаёт поля по выбранному языку, например: `titleRu`, `titleEn`, `titleKg` или один `title` в зависимости от заголовка `Accept-Language`. Фронт просто показывает пришедшее значение.

2. **Ключи перевода в БЗ**  
   В БД хранится ключ (например `book.title.123`), API отдаёт его. На фронте: `t(book.titleKey)` — тогда перевод подтягивается из `resources`. Нужно синхронизировать ключи в БЗ и в `resources`.

3. **Сервис перевода**  
   Для произвольного текста из БЗ (без ключей) — отдельный сервис перевода (например API перевода). Не часть i18n.

Итог: переводить контент из БЗ через i18n **получится**, если либо API отдаёт уже переведённые значения, либо из БЗ приходят ключи, которые есть в `resources`.
